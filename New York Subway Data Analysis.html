The statistical test that I used to analyze the NYC Subway data is the Mann-Whitney-U Test. This test is chosen after taking a look at the histogram of the number of entries in NYC subway in raining and not raining conditions. The histograms shows that the distributions of the number of entries does not follow a normal distribution. The Mann-Whitney-U test is sufficient when we want to see if the difference between too samples is statistically significant without assuming the normality of population distribution or taking large sample size to approximate normality.

I used the two-tail P values because we are guessing that the number of entries will be different when it rains. The null hypothesis is that when it rains, the number of people riding the subway will not be significantly different than when it doesnt rain. That means if we were to count every person entering the subway when it rains and when it doesnt rain, we will always find approximately the same number. Our alternate hypothesis is that there will be a difference.

In mathematical term: Null hypothesis: R - NR = 0 Alternate hypothesis: R - NR != 0

We will perform this test using significance level of = 0.05. This makes our p-critical value smaller than 0.025 or larger than 0.025, which means that if our p value is within these ranges, we can reject the null hypothesis.

The result I got from Mann-Whitney-U test:

With rain mean = 1105.446 Without rain mean = 1090.279 U value: 1924409167.0 P value: 0.0249

These results mean that assuming the total number of people riding subway when it rains is not different from the total number of people riding subway when it doesnt rain, the chance of selecting a sample difference of 15.167 (1105.446-1090.279) from the sampling distribution of sample means is only 0.0249 (2.49%), which is less than our threshold of 2.5%. Because of this, we can conclude that the difference in ridership does not happen by chance, but rather by an actual difference in the total number of people riding subway when it rains versus when it doesnt rain. However, the p-value is very close to the p-critical value, which suggest that there is only a slight difference between the means.

def linear_regression(features, values):
    
    # Perform linear regression given a data set with an arbitrary number of features.
    
    features = sm.add_constant(features)
    model = sm.OLS(values,features)
    result = model.fit()
    intercept = result.params[0]

We have pretty low R-squared, which suggests that overall, the model did not do a good job in predicting the subway ridership. One interesting thing to note is that while the Mann-Whitney U test suggests that there is a significant difference in subway ridership between rain and non-rain condition, the regression suggests that rain is not an important predictor (since the p-value for its coefficient is quite large). How can this be? As we look in the graph between mean entries per hour and rain, we can see that there is indeed a difference between mean entries when it rains vs when it doesn't rain, but rain is a poor predictor for future ridership. The rain variable groups the entire data into two categories, thus it doesn't provide much information to explain the variation in entries per hour (which is a continuous quantity). Therefore, while being a good categorizer (we know that when it rains the number of people riding subway will be different), but we do not know for sure how different.

Conclusion¶

From my analysis, it is shown that the number of people ride subway when it's raining is significantly different than when it's not raining. When we perform the Mann-Whitney-U test, we found out that the difference is statistically significant with p < 0.025. This has led me to conclude that the ridership when its raining is significantly different, although we do not know if the number of subway riders will increase or decrease.

Some shortcomings from the data are that the data doesnt include enough features for an effective prediction. In reality, people rely on a lot of factors more than weather to decide whether or not to use the subway. Some variables that can be added to improve our analysis include the geographical area that each subway routes cover, a dummy variable which indicates if there are any special events going on near one of the stations, or the amount of cars purchased last year to predict the potential traffic. All of these variables would add much more insight and predictive capability in our model.

One of the shortcomings of our analysis is the low R-value and rounding of data. Our R value is only 3%, which means that only 3% of ridership is explained using the variables in our model. Part of the problem is that our model includes quite a lot of categorical variables such as rain, fog, and hour. The lack of variation in this variable makes it difficult to formulate predictions using quantitative regression.

Furthermore, the excessive rounding in raw data also destroy the variance in our data. A lot of data points overlap each other, which makes it hard to both visuallize the data and conduct a meaningful regression analysis. I would suggest rounding the data to three decimal places. Without these drawbacks, I believe we can draw much more insight from the NYC subway data.
